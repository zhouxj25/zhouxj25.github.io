1. Policy定义

是用来定义一个class或class template的接口，该接口由以下项目的之一或全部组成：内隐型别定义(如typedef int * pointer)、成员函数和成员变量。Policy有点类似设计模式中的Strategy,只不过Policy更注重编译器，而Strategy更注重运行期。
基于策略(Policy)的类设计是将templates和多重继承组合起来，这样可以产生程序库中的“设计元素”。Policy由templates和多重继承组成。一个class如果使用了Policy,就称其为host class,那是一个拥有多个template参数的class template,每一个参数代表一个policy.host class所有的机能都来自policies,运作起来就像是一个聚合了数个policies的容器。
